# Multi-Domain Airline Optimization Pipeline

This repository contains a production-style Python analytics workflow for airline optimization.
The pipeline integrates:
- Flight operations
- Weather data
- Airport traffic/congestion
- Fuel prices
- Passenger sentiment

## Folder Structure

- `/raw_data` - Synthetic raw CSV files generated by the pipeline
- `/processed_data` - Cleaned and engineered datasets ready for analytics
- `/exports` - Power BI-ready star schema CSV exports
- `/reports` - Executive summary report
- `/models` - Saved ML models and scalers
- `main_pipeline.py` - Orchestration entry point

## How to Run

1. Install dependencies:
   - `pandas`
   - `numpy`
   - `scikit-learn`
   - `joblib`
   - `vaderSentiment`

2. Run:
   ```bash
   python main_pipeline.py
   ```

## Output Files

- `processed_flights.csv` - Cleaned flight operations dataset with KPIs
- `processed_sentiment.csv` - Enriched passenger sentiment scores
- `Fact_Flights.csv` and `Fact_Delays.csv` - Fact tables for Power BI
- Dimension CSVs in `/exports`
- `Executive_Report.txt` in `/reports`

## Star Schema Relationships

- `Fact_Flights` links to `Dim_Date` by `DateID`
- `Fact_Flights` links to `Dim_Route` by `RouteID`
- `Fact_Flights` links to `Dim_Airport` by `AirportID`
- `Fact_Flights` links to `Dim_Weather` by `WeatherID`
- `Fact_Flights` links to `Dim_Fuel` by `FuelID`
- `Dim_Sentiment` holds review-level sentiment records and can be joined to the flight fact by airport and date for trend analysis

## Notes

- The pipeline includes modular preprocessing, analytics, sentiment scoring, ML, reporting, and export steps.
- Power BI dashboards are not generated by this repository.
